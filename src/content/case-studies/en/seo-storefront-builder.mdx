## Context

The interesting part of this project was not a single storefront. It was a builder that generated storefront pages from internal product data, which made SEO output a system problem rather than a page-by-page content task.

## What I Worked On

- interface development
- GraphQL and Apollo-heavy product work
- SEO-related generation logic

## SEO Layer

- automated metadata generation
- Open Graph output
- partial implementation of dynamic e-commerce structured data
- direct reuse of internal product information as the source for markup

## Why It Mattered

The point was to reduce manual SEO work. If the product data already existed in internal systems and powered storefront cards, the same data should also power the markup layer. That made the builder more scalable and reduced the need for per-page manual filling.
