## Контекст

У этого проекта было необычное преимущество: он стоял на паузе. Это убирало большую часть release-pressure и позволяло сделать более крупную техническую перестройку аккуратно, а не латать все по ходу жизни продукта.

## Моя роль

- работал как senior frontend engineer в связке с одним middle-разработчиком
- участвовал в архитектурных решениях
- помогал определять новое направление фронтенда

## Технические изменения

- перевел кодовую базу с Vue 2 на Vue 3
- вынес frontend-работу из монолитной структуры
- перевел получение данных на выделенные API
- участвовал в настройке тестирования, code review и mentoring

## Результат

Поскольку продукт был на паузе, у команды было время на осознанную перестройку, а не на бесконечный patch-in-place. Поэтому этот кейс важен как история controlled modernization, а не как очередной emergency refactor.
